# 插件应用目录结构

## 标准目录结构

一个完整的 uTools 插件应用应该包含以下文件和目录：

```
插件应用根目录/
├── plugin.json          # 必需：插件配置文件
├── preload.js           # 必需：预加载脚本
├── index.html           # 必需：主界面文件
├── logo.png             # 必需：插件图标 (64x64像素)
├── index.js             # 可选：前端逻辑文件
├── index.css            # 可选：样式文件
├── assets/              # 可选：资源文件目录
│   ├── images/          # 图片资源
│   ├── icons/           # 图标资源
│   └── fonts/           # 字体文件
├── libs/                # 可选：自定义模块目录
│   ├── utils.js         # 工具函数
│   └── api.js           # API封装
├── docs/                # 可选：文档目录
│   ├── README.md        # 项目说明
│   └── api.md           # API文档
└── package.json         # 可选：npm包管理文件
```

## 文件说明

### 必需文件

1. **plugin.json**
   - 插件配置文件
   - 定义插件基本信息、入口文件、窗口设置等
   - 必须位于根目录

2. **preload.js**
   - 预加载脚本文件
   - 连接主进程和渲染进程
   - 可以调用 Node.js API 和 Electron 渲染进程 API
   - 必须保持代码清晰可读，不能压缩/混淆

3. **index.html**
   - 主界面文件
   - 插件的用户界面
   - 可以引用其他 CSS、JS 文件

4. **logo.png**
   - 插件图标文件
   - 推荐尺寸：64x64 像素
   - 支持 PNG、JPG 格式

### 可选文件

1. **index.js**
   - 前端逻辑文件
   - 处理用户交互和界面逻辑

2. **index.css**
   - 样式文件
   - 定义界面样式和布局

3. **assets/**
   - 资源文件目录
   - 存放图片、图标、字体等静态资源

4. **libs/**
   - 自定义模块目录
   - 存放工具函数、API封装等模块

5. **docs/**
   - 文档目录
   - 存放开发文档和使用说明

## 目录结构规范

### 文件命名规范

- 使用小写字母和连字符
- 避免使用特殊字符和空格
- 保持文件名简洁明了

### 路径引用规范

- 使用相对路径引用资源文件
- 避免使用绝对路径
- 确保路径在不同系统中兼容

### 模块组织规范

- 相关功能组织在同一目录下
- 模块之间保持低耦合
- 遵循单一职责原则

## 打包注意事项

1. **文件大小限制**
   - 单个插件包大小不超过 50MB
   - 合理压缩资源文件

2. **文件类型限制**
   - 只包含必要的文件
   - 避免包含开发工具配置文件

3. **代码可读性**
   - 预加载脚本必须保持清晰可读
   - 不能进行代码压缩/混淆

## 最佳实践

1. **模块化组织**
   - 将相关功能组织成独立模块
   - 使用 libs 目录存放工具函数

2. **资源管理**
   - 使用 assets 目录统一管理资源
   - 合理压缩图片资源

3. **文档维护**
   - 保持文档与代码同步更新
   - 提供清晰的 API 文档
